# Level Creator Guide: Dynamic Item Schema

This document outlines the schema for creating interactive items in **Screaming Fred**. Items are defined in `src/data/items.json`.

## Item Object Structure
```json
"item_id_here": {
  "name": "Display Name",
  "description": "What the user sees when looking at it.",
  "type": "food | tool | key | weapon",
  "onUse": {
    "action": "heal | unlock | damage_enemy | teleport | set_flag",
    "value": 5,
    "target": "target_tile_id",
    "flagSet": "story_flag_name",
    "consume": true,
    "successMessage": "Custom success text",
    "failureMessage": "Custom failure text"
  }
}
```

---

## Supported Actions

### 1. `heal`
Restores the player's health.
*   **value**: Amount of HP to restore.
*   **consume**: If `true`, the item is removed from inventory after use.

### 2. `unlock`
Unlocks a specific tile or path by setting a flag.
*   **target**: The ID of the tile to unlock (e.g., `exit_east`). The player must be on or adjacent to this tile.
*   **flagSet**: The flag to set to `true` in the room state.
*   **successMessage**: Message shown when unlocked.

### 3. `damage_enemy`
Damages an enemy. Useful for items like rocks, slingshots, or wands.
*   **target**: (Optional) The specific tile ID of an enemy. If omitted, the engine will **automatically target the closest enemy** in the room.
*   **value**: Damage amount.
*   **consume**: Use `true` for one-time weapons.

### 4. `teleport`
Instantly moves Fred to a different location or room.
*   **targetRoomId**: The ID of the destination room.
*   **targetPosition**: `{ "x": number, "y": number }` coordinates in the new room.

### 5. `set_flag`
Generic trigger for story events.
*   **flagSet**: Any flag name you want to trigger (e.g., `spoke_to_ghost`).
*   **successMessage**: Narrative text describing the event.

## Entities (Dynamic Objects)
Entities are defined in the `entities` array of a room. Unlike tiles, they can move and react to Fred's presence.

```json
"entities": [
  {
    "id": "patrol_snake_1",
    "name": "Patrolling Snake",
    "x": 3, "y": 1,
    "behavior": "patrol",
    "patrolPath": [{ "x": 3, "y": 1 }, { "x": 1, "y": 1 }],
    "hp": 3,
    "damage": 1
  },
  {
    "id": "shadow_stalker",
    "name": "Shadow",
    "x": 0, "y": 0,
    "behavior": "stalk",
    "detectionRange": 10,
    "hp": 5, "damage": 2,
    "loot": {
      "itemId": "item_sharp_rock",
      "chance": 1.0
    }
  }
]
```

---

## Enemy Loot & Rewards
When an enemy (static or entity) is defeated, it can drop rewards.
*   **loot**: An object with `itemId` and optional `chance` (0.0 to 1.0).
*   **defeatedFlag**: (Automatic) The engine sets `${id}_defeated` when health hits 0.

---

## NPC Interactions (Advanced)
NPCs can trigger events upon dialogue completion.
*   **onComplete**:
    *   `action: "give_item"`: Adds specified `itemId` to Fred's inventory.
    *   `action: "set_flag"`: Sets a global story flag.
*   **requiredFlag**: Gate dialogue stages behind story progress.
